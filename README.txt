📦 檔案清單
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

必要檔案：
✓ index.html              - 主頁面
✓ app.js                  - 應用程式邏輯
✓ manifest.webmanifest    - PWA 配置檔
✓ sw.js                   - Service Worker（離線支援）
✓ browserconfig.xml       - Windows Tiles 配置

圖示檔案（8個尺寸）：
✓ icon-72.png             - 72×72（Favicon）
✓ icon-96.png             - 96×96（標準）
✓ icon-128.png            - 128×128（中等）
✓ icon-144.png            - 144×144（Windows）
✓ icon-152.png            - 152×152（iPad）
✓ icon-192.png            - 192×192（Android 標準）⭐
✓ icon-384.png            - 384×384（高解析度）
✓ icon-512.png            - 512×512（最高品質）⭐

⭐ = Maskable 圖示（最重要）


🚀 安裝步驟
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

步驟 1：上傳檔案到 GitHub
────────────────────────────────────────────
1. 將所有檔案上傳到你的 GitHub 專案根目錄
2. 確保所有 icon-*.png 檔案都已成功上傳
3. 提交變更（Commit）

步驟 2：啟用 GitHub Pages
────────────────────────────────────────────
1. 前往專案設定（Settings）
2. 左側選單選擇「Pages」
3. Source 選擇「Deploy from a branch」
4. Branch 選擇「main」和「/ (root)」
5. 點擊「Save」
6. 等待幾分鐘讓網站部署完成

步驟 3：清除舊的快取（重要！）
────────────────────────────────────────────
⚠️ 如果之前已經安裝過 PWA，必須執行此步驟！

在 Android (Chrome)：
1. 長按主畫面的應用圖示
2. 點擊「解除安裝」
3. 開啟 Chrome 設定 → 隱私權 → 清除瀏覽資料
4. 勾選「快取的圖片和檔案」
5. 點擊「清除資料」

在 iOS (Safari)：
1. 長按主畫面圖示
2. 點擊「移除 App」
3. 設定 → Safari → 進階 → 網站資料
4. 找到你的網站並刪除資料

在 Desktop (Chrome/Edge)：
1. 按 F12 開啟開發者工具
2. Application → Clear storage
3. 勾選所有選項
4. 點擊「Clear site data」
5. Ctrl+Shift+R 強制重新載入

步驟 4：重新安裝 PWA
────────────────────────────────────────────
1. 訪問你的 GitHub Pages 網址
2. 等待「安裝應用程式」提示
3. 點擊「安裝」
4. 檢查主畫面圖示是否正確顯示


✅ 驗證圖示正確
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

正確的圖示應該有：
✓ 深綠色背景（#0d2419）
✓ 金色 "C" 字母
✓ 青色閃電符號
✓ 圓角設計

如果圖示不正確，請：
1. 確認所有 icon-*.png 檔案已正確上傳
2. 直接訪問 https://你的網址/icon-192.png 確認圖示
3. 清除瀏覽器快取
4. 解除安裝後重新安裝


🔧 疑難排解
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

問題 1：圖示還是舊的
解決：必須「解除安裝」PWA 後重新安裝，只清除快取不夠

問題 2：無法安裝 PWA
解決：確認 manifest.webmanifest 和 sw.js 已正確上傳

問題 3：圖示顯示為白色背景
解決：檢查 manifest.webmanifest 中的圖示路徑是否正確

問題 4：iOS 圖示是網站截圖
解決：確認 index.html 中的 apple-touch-icon 連結正確

問題 5：404 錯誤
解決：確認所有檔案都在專案根目錄，不是在子資料夾


📱 直接測試圖示
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

在瀏覽器中直接訪問以下網址來確認圖示是否正確上傳：

https://你的網址/icon-192.png
https://你的網址/icon-512.png
https://你的網址/manifest.webmanifest

如果看到 404 錯誤，表示檔案沒有上傳成功。


💡 重要提醒
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. PWA 圖示不會自動更新
   - 必須「解除安裝」後「重新安裝」
   - 這是所有 PWA 的標準行為

2. manifest.webmanifest 使用絕對路徑
   - 圖示路徑為 /icon-*.png（開頭有 /）
   - 這樣可以避免路徑問題

3. 清除快取很重要
   - 瀏覽器會快取舊的 manifest 和圖示
   - 必須清除快取才能看到新圖示

4. 無痕模式測試
   - 可以用無痕模式測試是否正確
   - 無痕模式不會使用快取
